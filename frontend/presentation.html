Bobibo - Architecture Presentation

Bobibo Product Store

A "No-JS" Framework E-Commerce Solution

Built with Go, HTMX, and PostgreSQL.

Infrastructure & Routing

Docker Services

We use Nginx as a Reverse Proxy to route traffic between the user and our Go backend.

  • Frontend (Port 80): Nginx serves static HTML.
  • Backend (Port 8000): Go API handles logic.
  • Proxy: /products requests are forwarded to Go.

Nginx Config (nginx.conf)

upstream backend_app {
    server backend:8000;
}

server {
    listen 80;

    # Static Files
    location / { root /usr/share/nginx/html; }

    # API Proxy
    location /products {
        proxy_pass http://backend_app;
    }
}

Flow 1: Browsing Products

We use HTMX to fetch HTML fragments based on Query Parameters.

Frontend (index.html)

Clicking a button sends a GET request with filters (e.g., `filter=smartphone`).

<!-- Fetch smartphones, offset 0, limit 10 -->
<button
    hx-get="/products?filter=smartphone&offset=0&page-size=10"
    hx-target="#product-view-area"
    hx-swap="outerHTML">
    View Smartphones
</button>

<div id="product-view-area">
    <!-- Results appear here -->
</div>

Why this works

  1. State: The URL holds the state (`?filter=smartphone`).
  2. Target: HTMX swaps only the content inside #product-view-area.
  3. Speed: No full page reload needed.

Flow 1: The Backend Handler

The Go backend parses the query parameters and renders a template.

func (dbc *DBContext) HandleGetProducts(c *gin.Context) {
    // 1. Parse Query Parameters (e.g. ?filter=smartphone)
    offset, _ := strconv.Atoi(c.DefaultQuery("offset", "0"))
    filterValue := c.Query("filter")

    // 2. Query Database (GORM)
    var products []models.Product
    dbc.DB.Where("category = ?", filterValue).Find(&products)

    // 3. Render HTML Template
    // We send "productsTemplate.html" filled with data
    c.HTML(200, "productsTemplate.html", products)
}

Key Fix: We use c.Query() instead of c.Param() to correctly handle the URL parameters sent by HTMX.

Flow 2: Adding to Cart

We use a persistent UUID for the user session to manage their cart.

The Template (productTemplate.html)

The server renders this button for every product.

<div class="product-box">
    <h3>{{.Name}}</h3>
</div>

<input type="hidden" name="product_id" value="{{.ID}}">

<!-- POST to specific User Cart ID -->
<button
    hx-post="/cart/a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11/add/{{.ID}}"
    hx-include="[name='product_id']"
    hx-target="#cart-view"
    hx-swap="innerHTML">
    Add to Cart
</button>

The Target (Cart Sidebar)

When the button is clicked, HTMX updates just this sidebar.

<div id="cart-view"
     hx-get="/cart/a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11"
     hx-trigger="load, cartUpdated from:body">
    <p>Loading cart...</p>
</div>

Data Persistence

We use PostgreSQL with a robust schema. The Products table uses ENUMs for strict categorization.

CREATE TYPE category AS ENUM ('smartphone', 'tablet', 'accessory', ...);

CREATE TABLE products (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    price DECIMAL NOT NULL,
    category category NOT NULL,
    attributes JSONB -- Flexible JSON for specs (color, storage)
);

CREATE TABLE carts (
    user_id UUID PRIMARY KEY REFERENCES users(id),
    item_id INTEGER REFERENCES products(id),
    quantity INT DEFAULT 1
);
